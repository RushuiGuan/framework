ICacheKey

A cache key is made of the prefix section and the value section. Each section of the key always ends with ':'.
The prefix section has a fix text and the value section can change or be empty. When the value is empty, the value section would be omitted. For example: mk:123: would be a key with prefix "mk" and value "123", mk: would be a key with prefix "mk" and no value.
